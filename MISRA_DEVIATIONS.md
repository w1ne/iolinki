# MISRA C:2012 Deviations

This log records approved deviations from MISRA C:2012 for the `iolinki` codebase.

## How To Use
- Add one entry per deviation.
- Link to the exact file and line.
- Include rationale and risk assessment.
- Specify whether the deviation is temporary or permanent.

## Template
```
ID: MISRA-YYYY-NNN
Rule: <MISRA C:2012 Rule>
Location: <file path>:<line>
Justification: <why this deviation is required>
Risk: <impact / mitigation>
Scope: <local | module | global>
Status: <temporary | permanent>
Reviewer: <name/date>
```
